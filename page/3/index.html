主要是进阶问题：如果二叉搜索树经常被修改（插入/删除操作）并且你需要频繁地查找第 k 小的值，你将如何优化 kthSmallest 函数？

这就涉及到了一些数据结构的知识，二叉搜索树是为了解决链表查找难而数组插入删除复杂度高的问题而产生的。但是为了维持住二叉搜索树的结构（左小右大），每次插入要先找到应该插入的地方。而且为了防止二叉树退化为链表，还要把插入后多余的子节点通过旋转重新恢复成平衡二叉搜索树。查找和插入的复杂度均为O(logn)。

所以简单来讲，对于插入删除以及查找都有要求的情形使用平衡二叉树就很好了。

如果还想要优化的话，可以从第K小的值下手。如果k的值是固定的，我们可以把构造一个B树，每一个节点的大小就是k个数。那么我们每次查找只需要读取根节点中存放的数组的最后一位就行，复杂度为O(1)。插入的话稍微复杂一点，但也可以在复杂度O(logn)范围内完成。

思路

不管怎么样，每遇到一个问题，先把暴力法的解法想出来，这样在面试的时候也不至于完全答不上来。你至少可以退而求其次地回答一个次优解。

暴力法显然就是三个for循环，一层一层地固定好三个数，再来检查是不是和为0。复杂度为 $O(n^3)$ 。如果我们要优化一定是掌握了这些数字的某些规律可以让我们少比较一些组合(也就是剪枝)。

不妨先把这些数字排序，那么三数之和一定是大中小三个数字组成，用h，m，l三个小写字母表示大，中，小的下标，大写字母H，M，L表示其代表的值。那么此时因为三个数我们有了排序，那么固定起来也有规律得多，不妨固定l和m，另m=l+1，l从0开始遍历。

有一个要注意的是不可以出现重复的三元组，也就是说一旦遇到一个三元组符合条件就可以不继续遍历h了，因为固定了l和m，再遍历获得的三元组肯定是一样的。另外，如果有两个数和以前相等，那么也要排除掉。

经典算法-LRU缓存机制

题目

原题

运用你所掌握的数据结构，设计和实现一个  LRU (最近最少使用) 缓存机制。它应该支持以下操作： 获取数据 get 和 写入数据 put 。

获取数据 get(key) - 如果密钥 (key) 存在于缓存中，则获取密钥的值（总是正数），否则返回 -1。
写入数据 put(key, value) - 如果密钥不存在，则写入其数据值。当缓存容量达到上限时，它应该在写入新数据之前删除最近最少使用的数据值，从而为新的数据值留出空间。

进阶:
你是否可以在 O(1) 时间复杂度内完成这两种操作？

思路

LRU机制是什么

举个例子，你现在有一排书，按照你上一次看他们的时间的远近从左到右排列，用A B C D表示（最右边的最新）。这个书架的大小是可以放5本书。
当你新买了一本书E的时候，就可以把书放在最右边，也就是D的后面。此时书架的排列是A B C D E。
此时你又买了一本新书F，但是书架已经满了，那么你就把最旧的书A抽出来，留出空位把F插入E的后面。此时书架的排列是B C D E F。
如果你有一天想看C书，那么你就要抽出来看完以后插入F的后面。此时的书架排列是B D E F C。

思想其实很简单易懂，这个算法一般都是用来做缓存处理的。因为计算机的内存大小有限，当缓存不够的时候，就可以用这个算法来决定把缓存里哪一个数据删掉。从而让程序执行每一次都可以缓存命中，加快程序执行速度。比如操作系统里的页置换算法。

怎么实现

高效率的实现无非就是一个构造数据结构的问题。这个算法需要插入，查找和删除的复杂度都为O (1)，而且要有大小排序。

  1. 排序可以用数组，但是插入的复杂度就不是O(1);
  2. 插入要O(1)可以用链表，但是查找的复杂度就不是O(1);
  3. 查找要O(1)的就会想到哈希表，但显然哈希表的存储方式不足以支撑排序的要求。

显然，没有一个现成的数据结构满足我们的要求，只能是自己组装。

  1. 首先查找用哈希表，存储一个指针指向一个对象，那么就可以扩充存储方式了。
  2. 然后我们把这个对象用双向链表的方式组织，加一个头加一个尾，有了排序特征的同时也保证了插入效率。

数据结构笔记

二叉树备忘录

概念解释

  1. 深度：边的条数，是以一个结点来讲的。
  2. 高度：整个树的最大深度，从叶结点往上数
  3. 等价二叉搜索树：若两颗搜索树的中序遍历序列相同，就说它们彼此等价。
  4. 数据局部性原理：（包含时间局部性和空间局部性）
    • 刚刚被访问的数据很有可能不久就会被再次访问
    • 将被访问的下一元素，极有可能就处于不久之前被访问的某个元素附近。

大纲介绍

  1. 为什么要引入二叉搜索树？
    因为线性结构要么静态查找很方便，动态删除插入不方便（比如序列）。要么动态插入删除方便，静态查找不方便（比如list）。于是就引出二叉搜索树这种半线性结构，在这两种操作之间达到均衡。
  2. 为什么要引入平衡二叉树？
    因为二叉搜索树的形态很多，最坏的时候可以退化成一个序列。这样就毫无意义，因为很多操作的时间复杂度都与树高度的线性相关。于是为了保证树的高度不会太坏,我们引入一个平衡因子的指标。从而定义平衡二叉树。理想状态下当然是树的高度达到$\lfloor log_2 n \lfloor $也就是完全二叉树。
  3. 为什么引入适当平衡？
    完全二叉树的要求未免太过严格，为了维持住这种形态所带来的重构的开销很有可能得不偿失。因此我们引入适当平衡——将树的高度不是绝对的等于$\lfloor log_2 n \lfloor $，而是渐进意义下的不超过$ O(log_2 n) $。于是,我们就可以介绍这一类型的树了：包括AVL，伸展树，红黑树，KD-树都是这一类型的变种。至于鼎鼎有名的B-树其实是完全二叉树的一种，因为她的外部结点的高度永远是一致的。
  4. 那么各种树的优缺点和解决了什么问题呢？

详解

二叉搜索树

  1. 要求：
    • 除了叶结点，每个结点最多2个子结点
    • 左结点比其父结点要小，右结点比其父结点要大
  2. 特点：
    • 这种规则的引入实际上就给搜索提供了遍历，本质上变成了一种二分法的搜索。
    • 另一方面，插入也变得简单。因为它的底层实现是一个链表，从而就不需要像序列一样有大规模的拷贝。

（高度）平衡二叉树–>B-树

  1. 思想就是把一个结点变为一个对象序列。用来解决数据量比较大的查找。
  2. 背后的原理：
    • 不同容量之间的读写速度是不一样的，内存里读取和磁盘里读取大概相差五个数量级
    • 在磁盘里读1b和读1kb的时间是差不多的，所以我们尽量一次多读一点。利用缓存实际上可以加快我们的速度
    • 对于数据量比较大，计算机是无法一次性把数据都放入内存的时候，于是在内存找不到的时候就去磁盘里找
    • CPU的数据读取并不是一个字节一个字节，现代操作系统下，为了加快速度，它找到了就会加载一个页大小（通常是4KB）的数据放入它的缓存体系中。
    • 于是如果按照普通的一个结点一个抽象的数据对象（比如10Bytes），由于我们的树是一个链表，是通过new来获取创建的，它在磁盘和内存中的分布都是没有规律的。那么它周围的结点（父结点，子结点等等)不一定会一起读进内存。于是我们就需要频繁的从磁盘里读。而根据数据局部性原理，这样就会大大降低效率。
    • 但是，如果我们把一棵合适规模的子树（一般就是一个页大小）放在一个序列里，序列在内存或者磁盘里是顺序存放的。于是CPU每次读一个结点就读到了整个彼此逻辑上有联系的对象，于是就减少了对磁盘的读取次数。
    • 而另一方面，CPU对内存的读取速度之快，使得在4kb数据下对序列进行遍历所带来的开销微不足道。由于数据量之小，在这里我们甚至不采用查找的一些高级算法。
  3. 特点：
    • 所以B-树的意义在于降低I/O次数，虽然比较次数不会降低，渐进意义上的时间复杂度也不会降低。但是因为I/O次数的降低使得整体查询时间的降低。
    • 外部结点是指叶结点的数值为NULl并不存在的孩子结点，B树的高度是相对于外部结点而言。引入外部结点的原因是可以对应于不同层次的存储结构。比如在内存的b树的外部结点是磁盘上的b树。
    • B树的外部结点的深度一概一样。
    • B树的阶次，m阶B树的内部结点
      • 上限：有不超过m-1个的关键码，不超过m的分支。
      • 下限：根节点是大于2个分支；其余内部结点的分支是大于 向上取整的（m/2)个
    • 插入 ->上溢 -> 分裂 唯有分裂到了根节点才会导致树的高度增加
    • 删除 ->下溢 -> 合并或者旋转 唯有合并到了根结点导致树的高度减小
    • 值得注意的是：对于插入，永远是对叶结点的插入，因为把非叶结点都可以看做是一个范围，一个值是无法和一个范围匹配的。只有叶结点表示的是一个独立的值。而删除的话，有可能遇到的删除是删除非叶结点里的数，思路是可以参考。B树的高度增加一定是由于分裂到根了，这样也就保证了b树的外部结点的深度一致。
    • N个内部结点就有N+1个外部结点

（适度）平衡二叉树

AVL树

  1. 要求：
    • 本身首先是一棵二叉搜索树。
    • 平衡条件：每个结点的左右子树的高度之差的绝对值（平衡因子）最多为1。
  2. 特点:
    • 如果因为插入删除导致平衡条件被破坏，我们可以通过单旋和双旋（这里涉及到等价二叉搜索树的概念，可以看下文参考）来重获平衡。
    • 这样就保证了二叉搜索树不会退化为一个序列。

伸展树

  1. 要求：
    • 伸展树则利用了数据局部性原理，对于二叉搜索树来讲，我们只需要将刚被访问的结点，及时地“转移”至树根附近m，就可以加速后面的操作。
  2. 特点：
    • 虽然它并没有一开始就要求平衡因子为1，但是不管哪一种初始结构，通过一个所谓的双层循环，不断地把所求的结点移到根结点。最后我们的树总是会处于一种高度平衡的状态。
    • 优点在于它体现了数据局部性的特征，并且通过双层伸展可以将当前查找结点上升到根节点的分摊成本降低至log(n),这和上面的avl的维持适度平衡需要的时间复杂度是差不多的。为什么单层伸展比双层伸展效率低呢？因为双层伸展把数的高度减小了

红黑树

  1. 要求
    1. 根结点和所有外部结点都是黑色
    2. 红色结点的孩子都是黑色——>也就是红色结点的孩子和父亲都是黑色
    3. 外部结点到根的黑色结点数量是一致的。
  2. 红黑树在很多情况下，不妨先把它看作一个4阶B树，对我们逻辑理解有很大帮助。比如
    • 一个红黑树通过提升就是一棵4阶B树。其黑高度就是一颗4阶B树的高度，也就是log(n)，于是考虑到红结点无法相邻，那么这个红黑树的高度最高也就是红结点间接插入黑结点中，也就是2倍log(n)的高度。由此可以证明红黑树的高度在渐进意义下也是log(n)
  3. 双红缺陷 与双黑缺陷
  4. 其最大优点是: 它使得树的重构复杂度降为常数。

重要结论

  1. 红黑树多用在内部排序，即全放在内存中的，微软STL的map和set的内部实现就是红黑树。B树多用在内存里放不下，大部分数据存储在外存上时。因为B树层数少，因此可以确保每次操作，读取磁盘的次数尽可能的少。在数据较小，可以完全放到内存中时，红黑树的时间复杂度比B树低。反之，数据量较大，外存中占主要部分时，B树因其读磁盘次数少，而具有更快的速度。
  2. STL里的map个set就是用的红黑树。而mysql里就是用的B+树。

练习一

这个实验是让我们处理do_pgfault函数的,而整个逻辑是:

产生页访问异常后 –> CPU把引起页访问异常的线性地址装到寄存器CR2中，并给出了出错码errorCode，说明了页访问异常的类型, OS把这个值保存在struct trapframe 中tf_err成员变量中。
中断服务例程会调用页访问异常处理函数do_pgfault进行具体处理
它根据从CPU的控制寄存器CR2中获取的页访问异常的物理地址以及根据errorCode的错误类型来查找此地址是否在某个VMA的地址范围内以及是否满足正确的读写权限
如果在此范围内并且权限也正确，这认为这是一次合法访问，但没有建立虚实对应关系。所以需要分配一个空闲的内存页，并修改页表完成虚地址到物理地址的映射，刷新TLB，然后调用iret中断，返回到产生页访问异常的指令处重新执行此指令。
如果该虚地址不在某VMA范围内，则认为是一次非法访问
首先页访问异常是CPU在找不到pa是的异常,这个异常是由硬件发出的. 那么我们代码可以不管.但是异常产生后,就是OS的责任来处理了.

操作系统理论

  1. 产生页访问异常的原因主要有：

    • 目标页帧不存在（页表项全为0，即该线性地址与物理地址尚未建立映射或者已经撤销)；
    • 相应的物理页帧不在内存中（页表项非空，但Present标志位=0，比如在swap分区或磁盘文件上)，这在本次实验中会出现，我们将在下面介绍换页机制实现时进一步讲解如何处理；
    • 不满足访问权限(此时页表项P标志=1，但低权限的程序试图访问高权限的地址空间，或者有程序试图写只读页面).

  2. 请描述页目录项（Pag Director Entry）和页表（Page Table Entry）中组成部分对ucore实现页替换算法的潜在用处。

  3. 如果ucore的缺页服务例程在执行过程中访问内存，出现了页访问异常，请问硬件要做哪些事情？

3-归纳推理问题

将问题简化为子问题。

题目1：screwy pirates

5个海盗分100个金币，分的规则如下：

  1. 让最年长的海盗来分，如果要超过半数的（这个例子里是3个）海盗同意，就按这个规则分，否则就把这个海盗丢到海里喂鲨鱼。
  2. 假设海盗都想要分得更多的钱，并且尽量杀死海盗。
    那么怎么分？

解答

首先我们把问题简化一下，2个海盗的话，用数字表示，数字越大，越年长。

  1. 1，2 ： 2先来分，由于他永远能得到自己的1票，也就是1/2的人支持。所以他可以选择把所有的金币都给自己。
  2. 1，2，3： 3先来分，这一轮3除了自己的一票，还需要争取到1票。争取1票的条件是把你杀死得到的金币要小于按照你的分法分到的金币。于是如果争取2的票，那么杀死3，2可以得到所有金币。那么3永远也得不到2的票。所以要争取1的票，那么杀死3，1一个金币也得不到，所以只需要给1 一个金币，1就会投你的票。
  3. 1，2，3，4： 4先来分，这一轮4也依旧是要争取1票。杀死4，3可以得到99个金币，2什么也得不到，1可以得到1个金币，争取3的票需要给100个金币给3。争取2的话，只需要给1个。争取1的话给2个。所以我们争取2的票，最后得到99个金币
  4. 1，2，3，4，5： 5先来分，这一轮需要争取额外的2票。考虑杀死5，4能得到99个金币，1，3一个金币也得不到，2能得到1个金币。所以争取1和3，也就是各给一个金币。自己得98个.

所以这个答案就是给1和3各一个金币，5给自己98个。
那么如果是10个，11个海盗呢？我们能找到这里得规律么？
首先我们看到每增加一个更年长的海盗，我们的思路其实就是先考虑把这个新增加的海盗杀死剩下的海盗各自能分多少钱。自己再稍微试一下6，7，8等。其实分发就是对于最年长的海盗是第2N+1个的分法是：第1，3，5，……到2N-1个海盗各给一个金币，剩下的给自己。对于最年长的海盗是第2N个的话，分法是:第2，4，6，……2N-2个海盗各自分1个金币，剩下的给自己。

可以用归纳法证明吗？很简单：

  1. 首先列出2个海盗和3个海盗的分法。符合我们的假设。
  2. 然后假设2N个海盗的分法和2N+1个海盗的分法。
  3. 然后尝试推导出2N+2和2N+3个海盗的分法。
    不多赘述。

题目2：Tiger and Sheep

100个老虎和1只羊，老虎可以吃草也可以吃羊（当然更喜欢吃羊），但是吃完羊的老虎自己会变成羊，又可以被其余老虎吃。假设老虎的第一要义是活下来，请问最初的这只羊会被吃掉吗?

解答

  1. 首先考虑1个老虎，显然这只羊会被吃掉。
  2. 两只老虎，老虎都会想，如果自己吃掉羊，另一只老虎就会把自己吃掉。所以老虎们都不会吃羊。
  3. 三只老虎，老虎会想，如果自己吃掉羊，剩下的两个会陷入情况2，自己就不会被吃掉。因此最先想到这个的老虎会吃掉那只羊。
  4. 四只老虎，老虎回想，如果自己吃掉羊，那么剩下的就会陷入情况3，从而自己会被吃掉。所以大家都不会吃羊。

规律显而易见，如果老虎的个数是奇数，羊就会被吃掉，如果是偶数，羊就不会被吃掉。

2-纸牌游戏

题目：

情形是一个赌场，从52张牌里抽4张（通过洗牌使得每一张抽取的概率是一样的），然后你从这四个里面选一张A，庄家再从剩下的三张里选一张B。如果A >B，则你赢。否则庄家赢。那么你赢的概率是多少？

解答

这里最直观的方法就是直接计算。首先因为每一张牌被抽取的概率是一样的，而且被选取的概率也是一样的，因此其实我们可以直接考虑从52张牌里选，去掉中间的抽取4张的细节。然后呢，我们一个个的来，如果A抽取的是2，则赢的概率是0，如果A抽取的是3，那么赢的概率是庄家抽的是2，即4/51.如果A抽取的是4，那么赢的概率是庄家抽取2和3，也就是8/51……
于是，0+1/134/51+1/138/51+1/1312/51+……1/1348/51=（4/（5113））（1+2+3+……12）=8/17.

但是还有一种解法。在此之前，先考虑一个问题，从一万张彩票（只有一张中奖）里选一张，前一个人选完的彩票后一个人就不能选了，请问每一个人抽到中奖的那一张的概率有区别吗？答案是没有，每个人中奖的概率是一样的，无论先后。所以这里也一样，虽然庄家在玩家后面抽，看起来像是一个条件概率，但其实总体来看，庄家和玩家一样，抽到每一张的概率是一样的。于是，我们考虑这个情形的三个结果：
C1：A>B
C2: A=B
C3： A<B

仔细体会1-投硬币问题的核心思想，同理可知P(C1)=P(C3)=x，P(C2)=y。 那么2x+y=1。然后我们看看P(C2) 可以算出来吗？其实就是134/523/51=1/17。于是x=16/17*1/2=8/17.

P(C2) 的计算过程是有13个相等的情况，每一种情况的概率是4/523/51。可能这里有人会问，不是说庄家和玩家抽到每一张的概率是一样的么？为什么这里不是4/524/52？因为我们前面的假设是先抽取的玩家抽到的牌是未知的，也就是没有固定，那么对于庄家来讲，它抽取到某一张的概率是一系列条件概率之和，而这里，我们的假设是玩家抽到的牌是已知的，是已经固定了的，于是它抽某一张牌的概率就只是一个简单的条件概率了。
