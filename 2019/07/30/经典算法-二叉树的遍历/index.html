二叉树的遍历

二叉树的遍历（中序，前序，后序，宽度优先）是基础，递归版本和非递归版本都要熟练掌握。递归版主要是要理解递归的思想，而非递归版主要是利用栈和队列来存储节点。

965. Univalued Binary Tree 这个题就可以囊括树的遍历算法。包括：前序遍历，中序遍历，后序遍历以及DFS和BFS。可以参考二叉树的递归与非递归遍历（前序、中序、后序)

中序遍历

原题：230. 二叉搜索树中第K小的元素

太基础了，直接上代码：

主要是进阶问题：如果二叉搜索树经常被修改（插入/删除操作）并且你需要频繁地查找第 k 小的值，你将如何优化 kthSmallest 函数？

这就涉及到了一些数据结构的知识，二叉搜索树是为了解决链表查找难而数组插入删除复杂度高的问题而产生的。但是为了维持住二叉搜索树的结构（左小右大），每次插入要先找到应该插入的地方。而且为了防止二叉树退化为链表，还要把插入后多余的子节点通过旋转重新恢复成平衡二叉搜索树。查找和插入的复杂度均为O(logn)。

所以简单来讲，对于插入删除以及查找都有要求的情形使用平衡二叉树就很好了。

如果还想要优化的话，可以从第K小的值下手。如果k的值是固定的，我们可以把构造一个B树，每一个节点的大小就是k个数。那么我们每次查找只需要读取根节点中存放的数组的最后一位就行，复杂度为O(1)。插入的话稍微复杂一点，但也可以在复杂度O(logn)范围内完成。

108. Convert Sorted Array to Binary Search Tree
因为这个序列是有序序列，最简单的就是中位数为根节点。然后左边部分的中位数为左结点，右边部分的中位数为右结点。递归版很简单，但是非递归版我们就可以用到栈。基本上树的很多操作都可以用到栈。
非递归创建的思路是先完成一个insert函数：

  1. 当树为空树，则当前插入点为root
  2. if(root>currentNode)插入左边，否则插入右边
  3. 上面是一个循环的过程，root是一个叶结点为止。

783. Minimum Distance Between BST Nodes
首先这个题目已经把输入放进了二叉搜索数里了，然后我们知道中序遍历是按照从小到大的顺序的，那么最小的差应该就在这个相邻的两个数之间。 最直观的思路就是先把树中序遍历一次放入一个数组，然后遍历数组得到最小差。
